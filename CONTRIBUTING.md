# Contributing to Supermango

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch from `main`
4. Make your changes
5. Submit a pull request

## Pull Request Process

1. All changes must go through a pull request — no direct pushes to `main`
2. PRs require at least 1 approving review before merge
3. Keep PRs focused — one feature or fix per PR
4. Write clear commit messages using conventional commits (`feat:`, `fix:`, `docs:`, `chore:`)

## Development

### Prerequisites

- [Claude Code](https://claude.ai/claude-code) with the following skills installed:
  - superpowers
  - gstack
  - context7
  - frontend-design

## Code of Conduct

Be respectful. Be constructive. Help each other build great things.
