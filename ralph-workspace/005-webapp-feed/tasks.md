# Tasks: 웹앱 통합 피드

## Task 1: 피드 API 및 데이터 조회
**Description:** 사용자의 워크스페이스에 속한 메시지를 최신순으로 조회하는 API를 구현한다. 무한스크롤을 위한 커서 기반 페이지네이션을 지원한다. status=complete인 메시지만 반환한다.
**Acceptance Criteria:**
- 사용자의 워크스페이스에 속한 완료된 메시지를 최신순으로 반환한다
- 커서 기반 페이지네이션이 동작한다
- 메시지와 함께 공유자, 채널, 태그, 이미지 URL 정보를 포함한다
**TDD Approach:** API 응답 형식 및 페이지네이션 테스트
**Validation:** API 호출로 정확한 데이터가 반환되는지 확인

## Task 2: 피드 카드 UI 컴포넌트
**Description:** 인스타그램 스타일의 피드 카드 컴포넌트를 구현한다. 카드뉴스 캐러셀(3장 스와이프), 공유자, 날짜, 불릿 3줄 요약, 태그 칩, 공유 버튼을 포함한다. 모바일 퍼스트로 디자인한다.
**Acceptance Criteria:**
- 카드뉴스 캐러셀이 좌우 스와이프로 동작한다 (인디케이터 ● ○ ○)
- 공유자 이름, 날짜, 불릿 3줄 요약이 표시된다
- 태그가 칩 형태로 표시된다
- 공유 버튼이 존재한다
- 다크 테마 디자인 (딥 네이비 배경, 노란색 액센트)
**TDD Approach:** 컴포넌트 렌더링 테스트, 캐러셀 스와이프 동작 테스트
**Validation:** gstack으로 모바일 뷰포트에서 카드 레이아웃 및 스와이프 확인

## Task 3: 무한스크롤 피드 페이지
**Description:** 피드 카드를 무한스크롤로 로딩하는 메인 피드 페이지를 구현한다. 스켈레톤 로딩 상태, 빈 상태 메시지를 포함한다.
**Acceptance Criteria:**
- 스크롤 시 다음 페이지를 자동 로딩한다
- 로딩 중 스켈레톤 카드가 표시된다
- 데이터가 없을 때 "아직 공유된 링크가 없어요" + 봇 초대 가이드가 표시된다
- 에러 시 재시도 버튼이 표시된다
**TDD Approach:** 스크롤 이벤트에 따른 데이터 로딩 테스트
**Validation:** gstack으로 무한스크롤 동작 확인
