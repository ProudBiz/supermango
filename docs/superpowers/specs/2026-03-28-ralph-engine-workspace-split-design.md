# Ralph Engine/Workspace Split

## Overview

Separate ralph's reusable engine (scripts, prompt) from per-run generated artifacts (specs, stories, tasks, progress) into two top-level directories: `ralph-engine/` and `ralph-workspace/`. Skills remain at `.claude/skills/ralph.*/` per Claude Code convention.

## Directory Structure

```
ralph-engine/              # Engine (reusable across runs)
  ralph-loop.sh            # Headless autonomous loop
  ralph-once.sh            # Single interactive iteration
  ralph-prompt.md          # Agent instructions template

ralph-workspace/           # Per-run artifacts (generated by planner)
  spec.md                  # Feature specification
  progress.json            # Global task state
  known-issues.md          # Stuck tasks/stories
  NNN-{story-slug}/        # One dir per user story
    story.md
    tasks.md
    log.md

.claude/skills/            # Skills (unchanged)
  ralph.coder/SKILL.md
  ralph.planner/SKILL.md
  ralph.reviewer/SKILL.md
```

| `ralph/ralph-once.sh` | `ralph-engine/ralph-once.sh` | Engine |
| `ralph/ralph-prompt.md` | `ralph-engine/ralph-prompt.md` | Engine |

| `ralph/progress.json` | `ralph-workspace/progress.json` | Artifact |

| `ralph/known-issues.md` | `ralph-workspace/known-issues.md` | Artifact |

## Script Changes

Shell scripts currently derive all paths from `SCRIPT_DIR` (the directory containing the script). After the split, artifacts are in a sibling directory:

```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_DIR="$(cd "$SCRIPT_DIR/.." && pwd)"
WORKSPACE="$REPO_DIR/ralph-workspace"
```

- Prompt file: `$SCRIPT_DIR/ralph-prompt.md` (stays with engine)
- Artifact files: `$WORKSPACE/spec.md`, `$WORKSPACE/progress.json`

Both `ralph-loop.sh` and `ralph-once.sh` get this update.

- `.claude/skills/ralph.reviewer/SKILL.md` — read paths to `ralph-workspace/`
